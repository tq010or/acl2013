Input plain text for geolocation prediction


Input a public Twitter user's screen name, e.g., @BarackObama

The system monitors Twitter Streaming API to identify users with geotagged tweets, then it crawls up-to 200 recent tweets from users and predicts the users' geolocations on basis of text data. The system currently makes use of tweet text, user declared location and user declared timezone information for location prediction. Due to limited network bandwidth, we further sub-sample Twitter users in the stream and it usually takes 30 seconds to crawl and predict users (depending on network speed as well).

    This is a deployment-driven research demo that predicts city-level geolocation on basis of text data. It has been part of my PhD work (in collaboration with my supervisors Paul Cook and Tim Baldwin) on processing social media data such as Twitter messages.
